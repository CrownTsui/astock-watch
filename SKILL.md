# A 股每日盯盘分析 Skill

对输入的 A 股股票执行技术面 + 资金面 + 消息面三维量化分析，输出交互式 HTML 报告。

## TL;DR 速查

```bash
cd /path/to/astock-watch
pip install -r requirements.txt   # 首次
python run.py                     # 默认自选股 → 盯盘全部标的
python run.py --symbol 600519     # 单只股票
python run.py --symbol 600519,000858 --open  # 多只 + 自动打开浏览器
```

| 用户说 | 执行 |
|--------|------|
| "盯盘" / "看盘"（无代码） | `python run.py`（分析 watchlist.txt 自选股） |
| "帮我看看 XXX" | `python run.py --symbol <代码>` |
| "分析 A 和 B" | `python run.py --symbol <A代码>,<B代码>` |
| "生成离线报告" | `python run.py --symbol <代码> --offline` |

## 执行流程

每一步完成后才能进入下一步；任一步命中失败分支时按失败模式表处理。

| 步骤 | 做什么 | 输入 | 输出 |
|------|--------|------|------|
| 1 | 解析用户输入的股票代码（支持逗号分隔、中文顿号、`--symbol`/`--file` 参数） | 用户消息文本 | 标准化 6 位代码列表 |
| 2 | 若无代码，回退读取 `watchlist.txt` 默认自选股池 | 代码列表（可能为空） | 最终待分析代码列表 |
| 3 | 逐只抓取行情（K线）、基本信息、资金流、北向持股、新闻公告 | 代码列表 | 每只股票的原始数据字典 |
| 4 | 🔴 **CHECKPOINT**：展示数据就绪摘要（每只股票的数据源、K线根数），若全部失败则终止 | 原始数据 | 用户确认继续 / 终止 |
| 5 | 计算技术指标（MA/MACD/KDJ/RSI/BOLL/WR/ADX）+ 三维分析 + 综合评分 | 原始数据 | 评分、操作建议、关键价位 |
| 6 | 运行回测校准（数据不足 80 根K线自动跳过） | 指标数据 | 回测结果（可选） |
| 7 | 🔴 **CHECKPOINT**：展示操作建议、仓位、关键价位与风险等级；用户确认后进入报告生成 | 评分结果 | 用户确认 |
| 8 | 渲染 HTML 报告（单股 + 多股对比汇总页），自动打开浏览器 | 全部分析结果 | 报告文件路径列表 |
| 9 | 🔴 **CHECKPOINT**：报告生成完毕，列出文件路径；询问用户是否需要打开浏览器查看或做进一步分析 | 文件路径列表 | 用户确认结束或继续 |

## 快速使用

```bash
# 安装依赖（首次）
pip install -r requirements.txt

# 不带任何参数：默认分析自选股（watchlist.txt 中的标的）
python run.py

# 单只股票
python run.py --symbol 600519

# 多只股票（逗号分隔）
python run.py --symbol 600519,000858,300750 --out-dir reports

# 指定分析日期 + 自动打开浏览器
python run.py --symbol 600519 --date 20260628 --open
```

报告输出到 `reports/astock_watch_<代码>_<时间戳>.html`，双击即可在浏览器打开。

> 💡 **默认自选股**：直接运行 `python run.py`（不传 `--symbol`/`--file`）时，读取 skill 根目录下的
> `watchlist.txt` 并分析其中全部标的。编辑该文件即可维护盯盘池。

## 失败模式与降级策略

### 行情抓取（核心，禁止模拟）

行情是分析的基石，**绝不使用模拟/伪造数据**。多数据源按优先级自动切换：

| 优先级 | 数据源 | 股票接口 | ETF/基金接口 |
|--------|--------|----------|-------------|
| 1 | 东方财富 | `ak.stock_zh_a_hist` | `ak.fund_etf_hist_em` |
| 2 | 新浪 | `ak.stock_zh_a_daily` | `ak.fund_etf_hist_sina` |

```
若 东财成功（K线 ≥ 30 根） → 使用东财数据
若 东财失败/不足           → 自动切换新浪源
若 新浪也失败              → 该标的跳过，不生成报告，输出 "✗ <代码>: 行情数据抓取失败（已尝试东财+新浪）"
```

### 通用失败分支

| 症状 | 一线修复 | 仍失败时 |
|------|----------|----------|
| 单个代码行情抓取失败 | 等待 2 秒后重试一次 | 跳过该标的，继续分析其余股票 |
| 全部代码行情均失败 | 检查网络连接 `curl -I https://finance.sina.com.cn` | 终止，提示用户检查网络或稍后重试（非交易时段数据可能延迟） |
| 资金流接口失败 | ETF/基金无个股资金流口径属正常；股票失败则资金面评分为中性 50 | 报告中标注"资金面：无数据" |
| 北向持股接口失败 | 2024-08 后官方停披露，自动跳过 | 报告中标注"北向：已停披露" |
| 新闻/公告接口失败 | 重试 1 次 | 消息面置空，评分为中性 |
| 回测数据不足（< 80 根K线） | 自动跳过回测模块 | 报告不显示回测图表 |
| akshare 库未安装 | `pip install akshare` | 提示用户安装后再运行 |
| 限流导致连接断开（RemoteDisconnected） | 自动退避重试 3 次（间隔 2s/4s/6s），相邻股票间隔 0.8s | 仍失败则走"单个代码失败"分支 |

## 分析维度

| 维度 | 内容 |
|------|------|
| 技术面 | 趋势（均线排列/ADX/斜率）、支撑压力、MACD/KDJ/RSI/WR/BOLL 信号、量价、形态识别 |
| 资金面 | 主力净流入力度与连续性、主力 vs 散户、北向资金倾向 |
| 消息面 | 近 7 日新闻/公告，金融情感词库打分，关键资讯提取 |
| 综合 | 三维加权评分、走势概率分布、操作建议、仓位、入场/止损/目标价位 |
| 回测 | 评分模型 IC 校准、分档收益、策略净值 vs 买入持有（严格无前视） |
| 对比 | 多股分析自动生成横向对比汇总页（可点击下钻各报告） |

## 数据来源（免费、无需 token）

- **行情**：akshare → 东方财富 / 新浪（双源互为兜底，应对单源限流）
- **资金流**：akshare → 东方财富个股资金流接口
- **北向持股**：akshare → 沪深股通接口（2024-08 后已停披露，自动降级）
- **新闻公告**：akshare → 东方财富资讯 + 巨潮资讯公告
- **禁止模拟数据**：所有数据源失败时直接报错跳过该标的，不生成虚假报告

## 禁忌清单（不要做的事）

| # | ❌ 不要做 | ✅ 正确做法 |
|---|----------|------------|
| 1 | 用模拟/随机数据生成报告（即使所有接口都失败） | 跳过无法抓取的标的，明确告知用户失败原因 |
| 2 | 在非交易时段假设当日数据已更新（当日 K 线可能为空或前一交易日数据） | 标注"数据截至最近交易日"，不做当日走势推测 |
| 3 | 直接给出投资建议（买入/卖出/仓位）而不附风险提示 | 每份报告末尾必须包含：⚠️ 本工具产出为量化模型机械计算结果，不构成投资建议 |
| 4 | 对 ETF/基金使用个股资金流接口（接口不支持，返回空或报错） | 识别 `is_fund=True`，资金面标注"ETF 无个股资金流口径" |
| 5 | 在用户未要求时分析 watchlist.txt 外的股票 | 严格按用户输入或 watchlist.txt 内容执行，不自行扩展标的 |
| 6 | 批量分析时无节制高频请求同一接口（触发东财 IP 限流封禁） | 相邻请求间隔 ≥ 0.8s，失败后指数退避重试 |
| 7 | 跳过回测不足提示直接展示策略净值（数据量不够时结果不可靠） | 回测数据 < 80 根K线时跳过并注明"数据不足，无法回测" |

详见 README.md（架构设计、自定义、每日定时运行）。

> ⚠️ 本工具产出为量化模型的机械计算结果，不构成投资建议。股市有风险，决策需谨慎。
